## 为什么

项目目前通过 MCP 和 Skill 包暴露同一套电费查询能力，但构建产物、运行路径和会话凭证处理相互交织。这会让项目更难安全打包，也会让助手调用不够稳定，并且不够贴合当前 MCP 结构化工具的使用方式。

## 变更内容

- 将 MCP 服务标准化为主要的结构化工具接口，提供明确的输入结构、输出结构、结构化结果、只读标注和一致的工具错误处理。
- 将 MCP 服务构建为零依赖单文件产物，允许部署时只复制 `mcp/server.cjs`、Node.js 运行时和本地凭证目录。
- 将 Skill 包标准化为可携带的助手调用包，明确运行路径规则、精简 `SKILL.md` 指引，并避免普通构建隐式复制凭证。
- 拆分开发构建、MCP 构建、Skill 构建和本地凭证打包命令，避免敏感本地状态被意外打进产物。
- 统一部署凭证目录层级：MCP 优先读取 `server.cjs` 同级 `.csg/`，Skill 优先读取 `cli.cjs` 同级 `.csg/`。
- 增加明确的凭证登录能力，让会话生成保持在本地，同时允许助手通过对话引导登录。
- 支持二维码登录和短信验证码登录的对话式流程，默认不在对话中暴露生成后的会话内容。
- 支持多个本地用户配置，让 MCP 和 Skill 可以查询多个南方电网登录用户下的电表数据。
- 记录 MCP 和 Skill 运行时的安全凭证存储与查找规则。

## 能力

### 新增能力

- `mcp-tools`：定义结构化 MCP 工具、零依赖部署产物、结构、结果形状、标注，以及账户、余额、用量和会话验证的错误行为。
- `skill-package`：定义可携带 Skill 包契约，包括文件、运行调用、会话查找、构建命令、脚本同级凭证目录和助手指引。
- `credential-login`：定义本地登录和会话生成行为，包括手动命令行登录和助手引导登录。
- `multi-user-profiles`：定义本地用户配置/会话注册表行为，用于管理多个登录用户并跨用户配置查询多个电表。

### 修改能力

- 无。

## 影响

- 受影响源码：`src/server.ts`、`src/cli.ts`、从现有 MCP 和 CLI 重复逻辑中抽取出的共享查询/用户配置/会话服务模块，以及 `src/login.ts`。
- 受影响构建文件：`package.json`、`tsconfig.json`、零依赖 `mcp/server.cjs`，以及生成的 `skills/china-southern-power-grid-stat/scripts/cli.cjs`。
- 受影响文档/产物：`README.md`、`skills/china-southern-power-grid-stat/SKILL.md`，以及保护生成或本地专用凭证文件的 `.gitignore`。
- 依赖可能包含较新的 MCP SDK 使用方式，以及实现运行时结构校验时需要的结构工具。
